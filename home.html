Version Control: Git

It helps us to keep track of the changes made in the code.

git init

This command sets up the toosl required to track all the changes made to the project.

A git project consists of three parts:

  1. Working directory

    This contains all the files that we usually work with.

  2. Staging area

    The list of all changes made to the working directory since "last save".

  3. Repository

    The list of all the changes made so for since the beginning of time.

General Workflow

Changes are made to the working directory. They are then staged to the staging area and permanently stored in the repository with the commit command.

git status

This shows the status of the git project, the number of staged files, unstaged files, changes made, etc.

git add filename

This command adds the file to the staging area. It means that we want to track the changes for this file.

git diff filename

It shows all the changes made to the file after it has been staged. These changes are the recent changed.

git commit

All the changes in the staging area are permanently stored in the repository.

Backtracking

If we want to go back to a previous state, we can use checkout

git show HEAD

This displays the git log for the HEAD commit and all the changes that were made for the HEAD commit.

git checkout commit

This is a read only command that helps us to view the working directory after a particular commit. The entire working directory changes. There is however no change in the repository.

git checkout commit filename

This helps us to go back to an older verison of the file. The changes are made in that particular file only.

git reset HEAD filename

This command helps us to unstage files from the staging area. The changes made in the working directory reamain as is.

git reset commit

This helps us to change the working directory to the state that it was after any particular commit.

Branching

git branch

Displays the list of all the branches and shows the one we are currently on.

git branch new_branch

This command creates a new branch with a given name.

git checkout branch_name

Used to move between branches. The working directory changes according to the commits.

git checkout -b new_branch

This creates a new branch with the given name and then checkout to the new branch

This is same as

git branch new_branch

git checkout new_branch

git merge experimental_branch

This merges the contents of the experimental_branch into the current branch we are present on.

If the merge is linera, then it is fast-forwarded. But if it is three-way merge, we have to do merge-conflict and then commit.
